# 📊 Superstore Sales Dashboard

An end-to-end interactive sales analytics dashboard built with **Streamlit**, **Pandas**, **Plotly**, and **Facebook Prophet**.

> 🔗 **Live demo:** [sales-dashboard.streamlit.app](https://sales-dashboard-xprqr9mthj84uxny3ygtcw.streamlit.app/)

---

## ⚠️ Python version

This project requires **Python 3.10, 3.11, or 3.12**.
Avoid Python 3.13+ for now — `pandas`, `pyarrow`, and `prophet` do not yet
ship stable wheels for the newest Python releases, and **Streamlit
Community Cloud itself only runs Python 3.9–3.12**, so a newer local
version won't match production anyway.

Check your version: `python3 --version`
Don't have 3.11? Install it:
- **macOS:** `brew install python@3.11`
- **Windows:** [python.org/downloads](https://www.python.org/downloads/release/python-3119/)
- **Linux:** `sudo apt install python3.11 python3.11-venv`

## Features

| Page | What it shows |
|---|---|
| 🏠 Home | KPI cards, category summary, raw data download |
| 📈 Overview | Monthly trend, sub-category revenue/profit, shipping modes, YoY comparison |
| 🗺️ Regional | US state choropleth, region heatmap, region detail table |
| 📦 Products | Top/bottom products, discount vs profit scatter, treemap, product search |
| 🔮 Forecast | 90-day forecast (Prophet, with automatic ARIMA fallback) + seasonality |

**Sidebar filters** (apply across all pages): date range · region · category · segment

## Quick start

**Option A — automatic setup (recommended):**

```bash
git clone https://github.com/faizamfairooz/sales-dashboard.git
cd sales-dashboard
bash setup.sh        # auto-detects Python 3.10/3.11/3.12 and installs everything
```

**Option B — manual setup:**

```bash
# 1. Clone the repo
git clone https://github.com/faizamfairooz/sales-dashboard.git
cd sales-dashboard

# 2. Create a venv with Python 3.11 specifically
python3.11 -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate

# 3. Install dependencies
pip install -r requirements.txt

# 4. Download the dataset
#    → https://www.kaggle.com/datasets/vivek468/superstore-dataset-final
#    → Save as data/superstore.csv

# 5. Run the cleaning script (once)
python data_cleaning.py

# 6. Launch the app
streamlit run app.py
```

## Deployment (Streamlit Community Cloud — free)

1. Push the repo to GitHub (make sure `data/superstore.csv` is in `.gitignore`)
2. Run `data_cleaning.py` locally once and commit `data/superstore_clean.parquet`
   (or add a small startup check in `app.py` that runs the cleaning step automatically)
3. The included `runtime.txt` tells Streamlit Cloud to use Python 3.11 — no extra config needed
4. Go to [share.streamlit.io](https://share.streamlit.io) → New app → connect your repo → Deploy
